## Overview

Content review ensures all text, copy, messaging, and content is accurate, professional, engaging, and error-free before production launch.

---

## 1. Text Content

### Copy Quality
- [ ] All text professionally written
- [ ] Tone of voice consistent
- [ ] Brand voice maintained
- [ ] No placeholder text (Lorem ipsum, etc.)
- [ ] No "Coming soon" or "TBD"
- [ ] No developer notes or comments
- [ ] All acronyms explained on first use

### Grammar & Spelling
- [ ] No spelling errors
- [ ] No grammar mistakes
- [ ] Punctuation correct
- [ ] Capitalization consistent
- [ ] No typos
- [ ] Sentences complete and clear

**Tools**:
```bash
# Use Grammarly, Hemingway Editor, or LanguageTool
# Manual review by content team
```

### Readability
- [ ] Reading level appropriate for audience
- [ ] Sentences not too long or complex
- [ ] Paragraphs digestible
- [ ] Bullet points used where appropriate
- [ ] Headers break up long sections
- [ ] Technical jargon minimized or explained

**Target**:
- Flesch Reading Ease: 60-70 (Standard)
- Grade Level: 8-10 (General audience)

### Consistency
- [ ] Terminology consistent throughout
- [ ] Product names spelled identically
- [ ] Company name consistent
- [ ] Feature names consistent
- [ ] Abbreviations used consistently

## 10. Brand Voice & Tone

### Voice Attributes
- [ ] Professional but approachable
- [ ] Technical but not intimidating
- [ ] Innovative and forward-thinking
- [ ] Confident but not arrogant
- [ ] Helpful and empowering

### Tone Guidelines
**Do**:
- Use active voice
- Address user directly ("you")
- Be specific and concrete
- Use examples
- Be encouraging

**Don't**:
- Use passive voice excessively
- Be vague or abstract
- Use unnecessary jargon
- Be condescending
- Overuse exclamation marks

### Examples
**Good**:
- "Visualize your data in 3D"
- "Connect your systems in minutes"
- "Automate workflows effortlessly"

**Bad**:
- "Leverage synergies for paradigm shifts"
- "Best-in-class solution provider"
- "Revolutionary game-changing platform"

## Content Review Process

### Step 1: Automated Checks
```bash
# Spell check
npm run spell-check

# Link checker
npm install -g broken-link-checker
blc http://localhost:5173 -ro

# Lighthouse content audit
lighthouse http://localhost:5173 \
  --only-categories=seo,accessibility \
  --output=html
```

### Step 2: Manual Review
1. Read all content aloud
2. Check for consistency
3. Verify links
4. Test CTAs
5. Review on multiple devices
6. Get fresh eyes (peer review)

### Step 3: Stakeholder Review
- [ ] Content team approval
- [ ] Marketing team approval
- [ ] Legal team approval (if required)
- [ ] Product owner approval
- [ ] Executive approval (if required)
